## 四、核心主流程（MVP 必须打通）

以下流程必须真实可跑通：

### 流程 1：文本输入并提交

1. 用户在输入框输入文本
2. 点击“提交”
3. 生成一条新的历史记录
4. 历史记录显示原文
5. 如果翻译开关开启，则自动生成译文
6. 输入框清空

### 流程 2：开启翻译并切换目标语言

1. 用户打开翻译开关
2. 用户选择目标语言
3. 之后新提交的内容自动翻译为该语言
4. 切换目标语言后，仅影响后续新提交内容，不影响历史记录

### 流程 3：重翻译历史记录

1. 用户点击某条历史记录的“重新翻译”
2. 弹出语言选择菜单 / 弹窗
3. 用户选择新的目标语言
4. 系统重新翻译该条记录
5. 只更新该条记录的译文和目标语言
6. 不新增新的历史记录

### 流程 4：语音录入

1. 用户点击语音按钮
2. 进入“录音中”状态
3. 再次点击按钮或点击停止后，结束录音
4. 进入“识别中”状态
5. mock 返回一段语音转写文本
6. 将转写文本填入输入框
7. 用户确认后点击提交，进入历史记录

### 流程 5：模型下载

1. 用户进入模型管理页
2. 点击某个未安装模型的“下载”
3. 模型状态切换为“下载中”
4. 显示下载进度条
5. 支持暂停 / 继续 / 取消
6. 下载完成后状态变为“已安装”

### 流程 6：使用未安装模型时提示下载

1. 用户开启语音录入，但语音模型未安装
2. 弹出下载提示
3. 用户可以跳转到模型管理页下载

4. 用户选择一个未安装的目标翻译语言
5. 弹出下载提示
6. 用户确认后可跳转下载或取消

## 五、需要实现的 mock service

请把服务层抽象清晰，后续方便替换成真实本地模型。

### 1）语音识别 mock service

接口能力：

- startRecording()
- stopRecording()
- transcribe()

行为要求：

- 录音时进入 recording 状态
- 结束后进入 transcribing 状态
- 最后返回一段 mock 文本
- 支持模拟失败

### 2）翻译 mock service

接口能力：

- detectLanguage(text)
- translate(text, targetLang)

行为要求：

- 自动识别语种
- 按目标语言返回对应 mock 译文
- 支持模拟翻译中状态
- 支持模拟失败

### 3）模型管理 mock service

接口能力：

- getModels()
- downloadModel(modelId)
- pauseDownload(modelId)
- resumeDownload(modelId)
- cancelDownload(modelId)
- deleteModel(modelId)

行为要求：

- 能模拟下载进度变化
- 能模拟安装完成
- 能模拟下载失败
